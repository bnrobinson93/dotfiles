The workflow, as the repo tells it

Reverse-engineered from ~/.dotfiles, July 2026. If any of this reads wrong, the config disagrees with your memory, and the config is probably right.

The shape of a day

Desktop: Arch + Omarchy (Hyprland, waybar, walker). Super+Enter opens a terminal via xdg-terminal-exec; Super+Alt+Enter drops straight into tmux. Waybar carries a TickTick task count, privacy indicators, battery. Webapps get first-class launchers (launch-or-focus-webapp).

Terminal: fish (zsh kept as a fallback), starship prompt, zoxide for jumping, mise for runtimes (node, go, rust, ruby, bun), fzf everywhere via fzf.fish.

Sessions: sesh + tmux, Ctrl+A prefix. sesh-select is the picker: live sessions first, ^f to reach cold project folders. Sessions blacklist agent workspaces (-ai, -exp, -orch) so pickers show human work only.

Editor: Neovim on LazyVim. Obsidian vault at ~/Documents/Vault for notes, hunk.nvim for diffs, Copilot + Copilot Chat in-editor.

Version control: jj-first, git-shaped at the edges

Everything meaningful happens in Jujutsu on colocated repos. Git exists for the outside world: GitHub, CI, coworkers.

  • ~55 jj aliases: jj tug to move bookmarks, revset aliases like ghbranch, a deliberate split between bare jj (clean log) and jj log (broad leaves view)
  • hunk as pager and interactive diff review; agents drive it through hunk session commands against the live daemon
  • lazyjj (local build) for stacked-PR management; ghpr/ghprs fish functions push bookmarks into PRs
  • SSH commit signing with stock ssh-keygen against on-disk keys; 1Password machines override via gitignored ~/.gitlocal / jj conf.d/z-local.toml
  • sesh startup commands run jj pull on session attach, so projects stay fresh without thinking about it

AI agents: three clients, one brain

Claude Code, Codex, and OpenCode all stow the same ai/ package: AGENTS.md as the shared entry point, VCS.md for jj discipline, WritingVoice.md for prose, CodeQuality.md for code. Caveman mode keeps chat terse; ponytail keeps solutions small.

  • herdr owns agent workspaces: jj-workspace plugin, tab titles, hunk autodiff. Agents work in -ai/-orch suffixed workspaces, invisible to sesh
  • Shared memory lives under ~/.claude/projects/.../memory/, readable by all three clients
  • Skills install via update-skills.sh: ponytail, hunk-review, teach, fabric, across all three agents in one pass

Provisioning

GNU Stow with a .stowrc that targets ~/.config by default; dot-local to ~/.local, zsh to ~, ai to each agent's config root, dot-ssh to ~/.ssh. Day to day, mise run stow / skills / update covers deployment and updates (topgrade behind the last one); install.sh bootstraps a fresh machine: pacman/apt basics, brew, mise, fisher, TPM, herdr, SSH keys, GitHub auth.

Ideas worth stealing from yourself

Shipped from earlier revisions of this list: shared sessionizer roots (~/.config/search-paths.txt feeds all four pickers), atuin deleted instead of half-wired, topgrade as the one updater, mise run stow/skills/update absorbing install.sh, and shell startup de-spawned (fish ~35ms: cached tool init, lazy completions, mise shims).

A weekly jj op log habit. You already lean on jj's safety. jj op log + jj undo recovers from anything an agent does in a workspace, which makes agent autonomy cheaper to grant. Worth a fish abbreviation.

The through-line: every tool here reduces ceremony between "I want to work on X" and working on X. The leftovers worth cleaning are mostly the previous generation of that same instinct.
